Appendix A

Requirement Explanation
1.1.1 Non-text Content: All non-text content that is presented to the user has a text alternative that serves the equivalent purpose. The Mango Phone Shop uses non-text content in two cases: The dynamic phone table and the logo of the site in the header. Both instances are used for decorational purposes and therefore do not need text alternatives.
1.3.1 Info and Relationships: Information, structure, and relationships conveyed through presentation can be programmatically determined or are available in text. The index.html file is well devided into a header, a body and a footer. Furthermore the tables are devided into table heads and table bodies, and have a caption tag. Also the "h2" tags and their children have a separate section. This all makes the relationships in the html file clear and readable.
1.3.2 Meaningful Sequence: When the sequence in which content is presented affects its meaning, a correct reading sequence can be programmatically determined. A meaningful sequence can be be programmitcally determied by making use of the divided sections in the html file
1.3.3 Sensory Characteristics: Instructions provided for understanding and operating content do not rely solely on sensory characteristics of components such as shape, size, visual location, orientation, or sound. The Mango Phone shop meets this requirement, for example the tables are refered to in the text above them. So there are clear instructions on where to find the information and what the information is about.
1.4.1 Use of Color: Color is not used as the only visual means of conveying information, indicating an action, prompting a response, or distinguishing a visual element. The web page uses contrasting colors for the text and the tables; a lighter background against a darker text. In the headers a darker background is used against a lighter color.
1.4.2 Audio Control: If any audio on a Web page plays automatically for more than 3 seconds, either a mechanism is available to pause or stop the audio, or a mechanism is available to control audio volume independently from the overall system volume level. As our webshop does not use any audio files it does not have to meet this requirement.
2.1.1 Keyboard: All functionality of the content is operable through a keyboard interface without requiring specific timings for individual keystrokes, except where the underlying function requires input that depends on the path of the user's movement and not just the endpoints. All the form elments can be reached by pressing the tab button on the keyboard. Also the form can be submitted or reset by respectively selecting the submit button or the reset button, and pressing enter on the keyboard.
2.1.2 No Keyboard Trap: If keyboard focus can be moved to a component of the page using a keyboard interface, then focus can be moved away from that component using only a keyboard interface, and, if it requires more than unmodified arrow or tab keys or other standard exit methods, the user is advised of the method for moving focus away. Using the tab and the shift+tab button all the functionalities on the webpage can be accessed, a keyboard trap is not present.
2.4.1 Bypass Blocks: A mechanism is available to bypass blocks of content that are repeated on multiple Web pages. The Mango Phone Shop has only one web page, and all the information is easaly reachable. Therefore it would not be necesary to meet this requirement
2.4.2 Page Titled: Web pages have titles that describe topic or purpose. Our web page is titled: "Mango Phone Shop". This is done in the header within title tags.
2.4.3 Focus Order: If a Web page can be navigated sequentially and the navigation sequences affect meaning or operation, focusable components receive focus in an order that preserves meaning and operability. The navigation buttons on top of the website allow the users to navigate directly to specific parts
2.4.4 Link Purpose (In Context): The purpose of each link can be determined from the link text alone or from the link text together with its programmatically determined link context, except where the purpose of the link would be ambiguous to users in general. All the links on the webpage have a dedicated text representation. So it is always clear where the link leads to.
